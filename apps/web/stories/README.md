# Stories

This folder contains feature documentation and user stories for Shorta Web.

## What's in here?

Each story document describes:
- **User journey** - Step-by-step flow from user perspective
- **Technical implementation** - Routes, APIs, data models
- **UI/UX decisions** - Design choices and rationale
- **Future enhancements** - Ideas for iteration

## Stories Index

- [001 - Generate Storyboard](./001-generate-storyboard.md) - Two ways to create director-style storyboards
- [002 - Conversational Storyboard Creation](./002-conversational-storyboard-creation.md) - Create storyboards from scratch using AI conversation
- [003 - Viral Pattern Analysis](./003-viral-pattern-analysis.md) - Analyze recent viral videos to incorporate proven patterns into storyboards

## Why document features as stories?

1. **Shared understanding** - Team alignment on what we're building
2. **Decision history** - Track why we chose certain approaches
3. **Onboarding** - Help new team members understand features quickly
4. **Planning** - Reference when building related features
5. **Iteration** - Compare current state vs proposed enhancements

## Format

Stories follow this structure:
```
# Feature Name

## Overview
Brief description

## User Journey
Step-by-step from user perspective

## Technical Implementation
Routes, APIs, data models

## UI/UX
Design decisions

## Future Enhancements
Ideas for iteration
```

## Contributing

When adding new features:
1. Create a new `.md` file in this folder
2. Document the user journey first
3. Add technical details second
4. Update this README index
